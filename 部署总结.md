### ✅ 4. 配置 JWT 验证
在 `SecurityConfig.java` 中配置：
- JWT 解码器连接到 auth-service 的 JWK 端点：`http://localhost:8081/.well-known/jwks.json`
- JWT 签发者验证：`http://localhost:8081`
- 公开端点配置：`/health`, `/actuator/**`, `/api/auth/**`, `/oauth2/**`, `/.well-known/**`

### ✅ 5. 实现核心功能类

#### SecurityConfig.java
- 配置安全过滤器链
- 设置 JWT 验证
- 定义公开和受保护端点

#### GlobalExceptionHandler.java
- 统一异常处理
- 返回标准化的错误响应格式

- JWT 认证过滤器

- 验证 JWT 并提取用户信息
- 将用户信息通过请求头传递给下游服务

#### HealthController.java
- 健康检查端点
- 网关信息端点

## 关键配置

### JWT 验证配置
```yaml
spring:
  security:
    oauth2:
      resourceserver:
        jwt:
          jwk-set-uri: http://localhost:8081/.well-known/jwks.json
          issuer-uri: http://localhost:8081
```

### 路由配置示例
```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: auth-service
          uri: http://localhost:8081
          predicates:
            - Path=/api/auth/**
          filters:
            - StripPrefix=2
```

## 已知问题

### 1. CORS 配置问题
- **问题**: 当通过网关访问 auth-service 时，遇到 CORS 错误
- **原因**: auth-service 本身的 CORS 配置与网关配置冲突
- **解决方案**: 需要调整 auth-service 的 CORS 配置或在网关中完全禁用 CORS

### 2. Spring Boot 版本兼容性
- **问题**: Spring Boot 3.5.9 与 Spring Cloud 2024.0.0 版本不兼容
- **解决**: 添加配置禁用兼容性检查：`spring.cloud.compatibility-verifier.enabled=false`

## 如何使用

### 1. 启动服务
```bash
# 先启动 auth-service（端口 8081）
cd auth-service
mvn spring-boot:run

# 在另一个终端启动 gateway-service（端口 8080）
cd gateway-service
mvn spring-boot:run
```

### 2. 测试端点
```bash
# 健康检查（公开）
curl http://localhost:8080/health

# 通过网关访问 auth-service（不需要 JWT）
curl http://localhost:8080/api/auth/health

# 访问需要 JWT 的端点（需要携带有效 JWT）
curl http://localhost:8080/api/users/profile \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```
